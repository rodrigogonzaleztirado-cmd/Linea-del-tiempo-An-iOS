Evolución de Android (2008 - 2025)

Historia y evolución del sistema operativo móvil más usado en el mundo

Android 1.0

API 1 - Septiembre 2008

Android 1.0 fue la primera versión comercial del sistema operativo desarrollada por Google y la Open Handset Alliance. Se lanzó oficialmente junto con el HTC Dream (T-Mobile G1). Esta versión incluía aplicaciones básicas como Gmail, Google Maps, YouTube y un navegador web basado en WebKit. Introdujo el sistema de notificaciones en la barra superior, una de las características que diferenció a Android desde sus inicios. No tenía nombre de postre oficialmente.

Android 1.5 Cupcake

API 3 - Abril 2009

Cupcake fue la primera versión en adoptar nombres de postres. Introdujo el teclado virtual en pantalla, lo que permitió dispositivos sin teclado físico. Agregó widgets en la pantalla de inicio, grabación y subida de video a YouTube, animaciones en transiciones y soporte para Bluetooth estéreo (A2DP). Marcó el inicio de la personalización real del sistema.

Android 1.6 Donut

API 4 - Septiembre 2009

Donut mejoró la compatibilidad con distintas resoluciones de pantalla y tamaños de dispositivo. Introdujo búsqueda universal (Quick Search Box), mejoras en Android Market y mejor gestión de batería. También añadió soporte para redes CDMA y mejoras en cámara y galería.

Android 2.0/2.1 Eclair

API 5-7 - Octubre 2009

Eclair introdujo soporte para múltiples cuentas de correo, Google Maps Navigation gratuito, fondos animados (Live Wallpapers), mejoras en cámara con flash y zoom digital, y soporte para Bluetooth 2.1. Fue una versión clave en la expansión internacional de Android.

Android 2.2 Froyo

API 8 - Mayo 2010

Froyo mejoró significativamente el rendimiento gracias al compilador JIT. Permitió compartir internet mediante hotspot Wi-Fi, añadió soporte para Adobe Flash y mejoró la instalación de apps en tarjeta SD. También incorporó control por voz mejorado.

Android 2.3 Gingerbread

API 9-10 - Diciembre 2010

Gingerbread rediseñó la interfaz con tonos oscuros y verdes característicos. Mejoró el teclado, soporte NFC, administración de energía optimizada y soporte para cámaras frontales. Fue una versión extremadamente popular y duradera.

Android 3.0 Honeycomb

API 11-13 - Febrero 2011

Fue la primera versión diseñada exclusivamente para tablets. Introdujo la interfaz Holo con tonos oscuros y azules, botones virtuales en pantalla y la Action Bar para acciones contextuales. También incorporó aceleración por hardware y mejor soporte para procesadores multinúcleo, optimizando el rendimiento en pantallas grandes.

Android 4.0 Ice Cream Sandwich

API 14-15 - Octubre 2011

Unificó teléfonos y tablets en una sola plataforma. Añadió desbloqueo facial, botones virtuales permanentes, la fuente Roboto y mejoras en el control del uso de datos móviles. También optimizó la multitarea y la experiencia visual del sistema.

Android 4.1-4.3 Jelly Bean

API 16-18 - Julio 2012

Mejoró significativamente la fluidez del sistema gracias a Project Butter, que permitió animaciones más suaves a 60 fps. Introdujo Google Now y notificaciones expandibles e interactivas, además de mejoras en accesibilidad y gráficos.

Android 4.4 KitKat

API 19-20 - Octubre 2013

Optimizado para funcionar en dispositivos con poca memoria RAM. Incorporó modo inmersivo de pantalla completa, integración por voz con “Ok Google” y mejoras en el rendimiento general y la gestión de memoria.

Android 5.0-5.1 Lollipop

API 21-22 - Noviembre 2014

Presentó Material Design, un rediseño visual completo con animaciones modernas. Reemplazó Dalvik por ART como entorno de ejecución, añadió soporte para 64 bits, cifrado por defecto y mejoras importantes en duración de batería.

Android 6.0 Marshmallow

API 23 - Octubre 2015

Introdujo permisos solicitados en tiempo real, el modo Doze para ahorrar batería cuando el dispositivo está en reposo y soporte oficial para lector de huellas digitales. También mejoró la gestión del almacenamiento.

Android 7.0-7.1 Nougat

API 24-25 - Agosto 2016

Añadió multitarea en pantalla dividida, respuestas rápidas desde notificaciones y soporte para la API gráfica Vulkan. También implementó actualizaciones del sistema sin interrupciones en dispositivos compatibles.

Android 8.0-8.1 Oreo

API 26-27 - Agosto 2017

Incorporó modo Picture-in-Picture, canales de notificación configurables y límites más estrictos para aplicaciones en segundo plano. Además, introdujo Project Treble para facilitar futuras actualizaciones del sistema.

Android 9 Pie

API 28 - Agosto 2018

Implementó navegación por gestos, batería adaptativa basada en inteligencia artificial y herramientas de bienestar digital para controlar el uso del dispositivo. También mejoró la seguridad del sistema.

Android 10

API 29 - Septiembre 2019

Eliminó oficialmente los nombres de postres. Introdujo modo oscuro global, mejoras avanzadas de privacidad y una navegación por gestos más completa. También mejoró el soporte para dispositivos plegables.

Android 11

API 30 - Septiembre 2020

Añadió burbujas de conversación, permisos de una sola vez para apps y grabador de pantalla nativo. Mejoró la gestión de dispositivos inteligentes y la privacidad del usuario.

Android 12

API 31-32 - Octubre 2021

Presentó Material You, permitiendo personalización dinámica de colores basada en el fondo de pantalla. Incluyó panel de privacidad, indicadores de uso de cámara y micrófono, y animaciones más fluidas.

Android 13

API 33 - Agosto 2022

Incorporó permisos específicos para notificaciones, mejoras en Bluetooth LE Audio y optimización para pantallas grandes. Reforzó la seguridad y el control de acceso a archivos.

Android 14

API 34 - Octubre 2023

Mejoró la accesibilidad permitiendo mayor tamaño de fuente, añadió soporte para imágenes Ultra HDR y reforzó restricciones para aplicaciones antiguas por seguridad.

Android 15

API 35 - 2024

Se enfocó en mejorar la multitarea en tablets y dispositivos plegables, además de optimizar el rendimiento y fortalecer la privacidad en segundo plano.

Android 16

API 36 - 2025

Integra funciones avanzadas de inteligencia artificial a nivel del sistema, mejora la eficiencia energética y amplía la compatibilidad con dispositivos plegables y tecnologías emergentes como realidad extendida.
